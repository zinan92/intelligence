# 翡翠信号图谱 (Jade Signal Atlas) 原型

## 简介

这是翡翠信号图谱的高保真中文仪表盘原型 — 面向业务负责人的产品决策支持系统。原型展示了如何将复杂的翡翠市场研究数据转化为清晰、可执行的产品线决策建议。

**核心设计原则：信号优先，判断次之，证据第三。**

这个原型是 intelligence 仓库的一个**新增产品层**，与现有的 `report.html` 评估界面平行存在，不取代它。`report.html` 面向数据分析和方法验证；翡翠信号图谱面向产品决策和业务行动。

## 如何查看原型

### 启动本地服务器

在 `intelligence` 仓库根目录下运行：

```bash
python3 -m http.server 8765 --directory dashboard
```

### 在浏览器中打开

打开浏览器访问：

```
http://localhost:8765/
```

或直接访问任何页面：
- 首页：`http://localhost:8765/index.html`
- 方向地图：`http://localhost:8765/direction-map.html`
- 产品线观察：`http://localhost:8765/product-line.html`
- 证据库：`http://localhost:8765/evidence.html`
- 方向详情：`http://localhost:8765/direction-detail.html?id=dir_001`

## 页面概览

### 首页 (index.html)

仪表盘总览，包含 7 个核心模块：
1. **实时异动榜** — 最近有哪些方向正在异动（信号优先）
2. **今日判断** — 聚合判断统计（值得跟/观察中/短热噪音/需要补证据）
3. **值得关注方向** — 当前重点关注的方向卡片
4. **风险提醒** — 需要注意的风险点
5. **最近证据** — 新增高价值证据
6. **产品线观察** — 四条产品线的状态快照
7. **14天变化** — 升温/降温/新出现的方向

### 方向地图 (direction-map.html)

所有追踪方向的完整列表与对比视图。支持按热度、置信度、名称排序，以及按判断状态筛选。每个方向显示：热度、置信度、判断状态、分类、受众适配、价格带。

### 方向详情 (direction-detail.html)

单个方向的深度分析页面，包含：
- **方向概览** — 当前热度、时间窗口、置信度、判断状态、受众适配、价格带
- **一句话建议** — 清晰的行动建议
- **产品线决策卡片** — 四条产品线（项链/吊坠、耳饰、戒指、手串/叠戴）的机会/风险评估和策略建议
- **为何异动** — 方向升温/降温的原因分析
- **风险警示** — 具体风险点和缓解措施
- **证据入口** — 支持该方向的代表性证据链接

### 产品线观察 (product-line.html)

按产品线视角看市场 — 翻转分析维度，从"方向→产品线"变为"产品线→方向"。展示：
- 四条产品线的整体状态（上升/高潜但不稳定/证据不足/当前人群更稳）
- 哪些方向增强该产品线，哪些方向削弱该产品线
- 每条产品线的机会等级和风险等级

### 证据库 (evidence.html)

证据探索界面，采用三栏布局：
- **左栏** — 筛选器（方向、产品线、时间范围、证据类型）
- **中栏** — 证据卡片列表（笔记/图片/评论）
- **右栏** — 选中证据的详细信息（为何引用、支持哪个方向、新鲜度评估、权重）

## 数据说明

### 真实数据 vs 模拟数据

| 内容类型 | 数据状态 | 说明 |
|---------|---------|------|
| 方向名称 | 真实 | 基于 jade 项目真实研究成果 |
| 判断状态 | 真实 | 基于实际评估方法 |
| 产品线分类 | 真实 | 基于翡翠行业实际分类 |
| 热度数值 | 模拟 | 用于原型展示的示例数值 |
| 置信度评估 | 模拟 | 用于原型展示的示例评级 |
| 证据条目 | 模拟 | 用于原型展示的示例证据 |

**注意：** 所有数据从 `dashboard/data/jade_dashboard.json` 加载。当前数据是为原型设计准备的丰富模拟数据，基于真实的翡翠研究内容结构。

### 判断状态词汇表

原型使用四个标准判断状态，颜色编码一致：

| 判断状态 | 含义 | 颜色 |
|---------|------|------|
| 值得跟 | 高置信度、高转化稳定性的方向，值得投入资源 | 绿色 |
| 观察中 | 有潜力但转化稳定性待验证，需要继续观察 | 蓝色 |
| 短热噪音 | 热度高但转化弱，或趋势不可持续 | 橙色 |
| 需要补证据 | 证据不足，需要更多数据支持才能做判断 | 灰色 |

## 技术架构

### 技术栈

- **纯静态 HTML/CSS/JavaScript** — 无构建工具、无 npm、无框架依赖
- **CSS 自定义属性** — 设计系统变量（颜色、间距、字体）统一管理
- **ES6+ 模块化 JavaScript** — 数据加载、组件渲染、导航逻辑分离
- **Python http.server** — 本地开发/测试服务器

### 设计系统

所有页面共享统一的视觉语言：
- **高级分析美学** — 介于奢侈品产品规划和市场情报之间的视觉风格
- **卡片式模块化布局** — 响应式网格系统
- **中文优化字体栈** — PingFang SC, Microsoft YaHei, Hiragino Sans GB 等
- **语义化颜色编码** — 判断状态、机会/风险、升温/降温使用一致的颜色映射
- **清晰的排版层次** — 标题、正文、辅助文本分级清晰

## 与现有 report.html 的关系

| 维度 | report.html (评估层) | 翡翠信号图谱 (产品层) |
|------|---------------------|---------------------|
| **受众** | 数据分析师、方法研究者 | 产品负责人、业务决策者 |
| **目的** | 验证方法有效性、展示算法输出 | 支持产品决策、指导业务行动 |
| **语言** | 英文（技术导向） | 中文（业务导向） |
| **信息密度** | 高（详尽数据展示） | 中（关键洞察突出） |
| **视觉风格** | 简洁技术报告 | 高级分析仪表盘 |
| **关注点** | 数据质量、方法验证 | 信号→判断→行动 |

**两者不冲突：** `report.html` 继续服务于方法验证和技术评估；翡翠信号图谱服务于产品规划和业务决策。它们从不同角度支持智能化内容分析。

## 开发约定

### 编码规范
- **语义化 HTML5** — 使用正确的标签表达内容结构
- **CSS 自定义属性** — 颜色、间距、字体集中管理
- **Vanilla JavaScript (ES6+)** — 无 jQuery、无框架
- **命名规范** — CSS class 使用 kebab-case，JavaScript 变量使用 camelCase
- **UTF-8 编码** — 所有文件使用 UTF-8
- **`<html lang="zh-CN">`** — 所有页面标记中文语言

### 关键约定
- **判断状态词汇一致性** — 所有页面必须使用相同的四个判断状态（值得跟、观察中、短热噪音、需要补证据）
- **颜色编码一致性** — 判断状态颜色在所有页面保持一致
- **导航栏一致性** — 所有页面共享相同的导航栏组件（通过 `js/nav.js` 自动注入）
- **卡片样式一致性** — 所有卡片使用相同的边框半径、阴影、padding
