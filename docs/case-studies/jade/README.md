# Jade Case Study

This case study captures what the first jade pack proved for the `intelligence` repo:

- jade needs a lean runtime config plus a richer strategy layer in docs
- sampling works best when it balances core jade accounts, adjacent-category accounts, and keyword discovery
- keyword discovery should stay split across direct jade demand, modern design language, material combinations, styling scenes, price intent, and adjacent categories
- scoring should reward image strength and cross-category resonance, not just raw engagement
- the pack can produce multiple downstream outputs without dragging category-neutral workflow rules into jade-specific docs

## What Moved Here

- sample-frame concepts
- scoring concepts
- the jade-specific rationale behind sample selection and trend judgment

## What Stayed in the Pack

- the reusable keyword list in `src/intelligence/projects/jade/keywords/seed_keywords.csv`
- the pack contract in `src/intelligence/projects/jade/config/project.yaml`
- the output templates in `src/intelligence/projects/jade/templates/`
